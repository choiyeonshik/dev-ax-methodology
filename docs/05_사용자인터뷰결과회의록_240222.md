# 사용자 인터뷰 결과 분석 회의록

**회의명**: 자동차 정비 예약 플랫폼 사용자 인터뷰 결과 분석  
**일시**: 2024년 2월 22일 15:00-18:00  
**장소**: 본사 회의실  
**참석자**: 김사업(대표), 박영업(영업이사), 정디자인(디자인팀장), 이기술(CTO), 외부 UX 리서처

## 1. 인터뷰 개요

### 1.1 인터뷰 대상
**일반 고객 인터뷰** (2월 12일-19일)
- 총 15명, 연령대 25-55세
- 차량 보유자, 최근 1년 내 정비 경험 있음
- 지역: 서울 8명, 경기 5명, 부산 2명

**정비소 운영자 인터뷰** (2월 13일-20일)  
- 총 10곳 (소형 5곳, 중형 3곳, 대형 2곳)
- 업력 5년 이상
- 지역: 서울 4곳, 경기 4곳, 부산 2곳

### 1.2 인터뷰 방법
- 개별 면담 (1시간/명)
- 반구조화 인터뷰
- 현재 정비 프로세스 관찰
- 프로토타입 사용성 테스트

## 2. 고객 인터뷰 주요 결과

### 2.1 현재 정비소 선택 기준
**우선순위 순**
1. **거리/접근성** (100% 언급): "집이나 직장에서 가까운 곳"
2. **지인 추천** (80% 언급): "믿을 만한 사람 추천"
3. **가격** (73% 언급): "너무 비싸지 않은 곳"
4. **과거 경험** (67% 언급): "한 번 가본 곳"
5. **온라인 리뷰** (40% 언급): "네이버 리뷰 참고"

### 2.2 주요 불만사항 및 페인포인트
**가격 투명성 부족 (93% 언급)**
- "정비 받기 전까지 정확한 비용을 모름"
- "부품값이 정말 이게 맞나 의심스러움"
- "추가 정비 권유 시 거절하기 어려움"

**진행상황 불투명 (87% 언급)**
- "차를 맡기고 나면 언제 끝날지 모름"
- "중간에 뭘 하고 있는지 궁금함"
- "완료 연락만 기다리는 답답함"

**예약 시스템 부재 (73% 언급)**
- "직접 가서 몇 시간 기다림"
- "전화로 문의해도 정확한 시간 못 알려줌"
- "바쁜 시간에 맞춰갈 수 없어 불편"

**소통의 어려움 (60% 언급)**
- "정비사와 직접 대화하기 어려움"
- "전문 용어 때문에 이해 어려움"
- "궁금한 것을 편하게 물어보기 힘듦"

### 2.3 원하는 서비스 기능
**필수 기능 (High Priority)**
- 사전 견적 시스템 (100%)
- 실시간 진행상황 확인 (93%)
- 온라인 예약 시스템 (87%)
- 정비소 비교 기능 (80%)

**선호 기능 (Medium Priority)**
- 정비 이력 관리 (73%)
- 리뷰 및 평점 시스템 (67%)
- 다양한 결제 방법 (60%)
- 정기점검 알림 (53%)

**추가 기능 (Low Priority)**
- 채팅 상담 (40%)
- AI 추천 (33%)
- 부품 온라인 구매 (27%)

### 2.4 사용성 테스트 결과
**프로토타입 첫 인상**
- "깔끔하고 직관적" (80%)
- "생각보다 간단해 보임" (73%)
- "믿을 만해 보임" (67%)

**예약 프로세스 피드백**
- 평균 완료 시간: 4분 32초
- 성공률: 87% (13명/15명)
- 주요 실패 지점: 차량 정보 입력 (복잡함)

**개선 요청사항**
- 차량번호 자동 인식 기능
- 정비 항목 설명 추가
- 가격 범위 미리 표시
- 취소 정책 명확히 표시

## 3. 정비소 인터뷰 주요 결과

### 3.1 현재 운영 방식 및 문제점
**예약 관리 방식**
- 전화 예약 (100%): "손님이 전화로 문의"
- 수기 스케줄 관리 (80%): "공책이나 화이트보드 사용"
- 단골 고객 위주 (70%): "새 고객 받기 어려움"

**주요 운영 어려움**
- **예약 관리의 복잡성** (100% 언급)
  - "전화 받느라 정비에 집중 못함"
  - "더블 부킹이 종종 발생"
  - "노쇼 고객 때문에 손해"

- **고객 소통 부담** (90% 언급)
  - "진행상황 문의 전화 많음"
  - "견적 설명하는데 시간 오래 걸림"
  - "추가 정비 승인받기 어려움"

- **신규 고객 확보 어려움** (80% 언급)
  - "마케팅할 시간과 방법 부족"
  - "온라인 홍보 방법 모름"
  - "젊은 고객층 접근 어려움"

### 3.2 플랫폼 서비스에 대한 기대
**필수 기능 요구사항**
- 자동 예약 관리 시스템 (100%)
- 고객 정보 자동 저장 (90%)
- 견적서 자동 생성 (90%)
- 진행상황 자동 알림 (80%)

**수익성 관련 우려**
- 중개 수수료 부담 (100% 언급)
  - "10%는 너무 높다" (70%)
  - "5-7% 정도가 적당" (50%)
  - "초기에는 무료였으면" (40%)

- 기존 고객 이탈 우려 (60% 언급)
  - "단골들이 플랫폼으로 갈까 봐"
  - "가격 비교 당하면 곤란"

### 3.3 플랫폼 참여 조건
**참여 의향**
- 적극 참여: 30% (3곳)
- 조건부 참여: 50% (5곳)  
- 소극적/관망: 20% (2곳)

**참여 조건**
- 초기 수수료 면제 또는 할인
- 충분한 교육 및 기술 지원
- 기존 고객 관리 방식 유지 가능
- 안정적인 신규 고객 유입 보장

### 3.4 시스템 기능 우선순위
**High Priority**
1. 예약 캘린더 관리
2. 고객 정보 관리
3. 견적서 생성 도구
4. 진행상황 업데이트 기능

**Medium Priority**
5. 재고 관리 (부품)
6. 매출 통계 및 분석
7. 고객 리뷰 관리
8. 마케팅 도구

## 4. 주요 인사이트 및 시사점

### 4.1 핵심 발견사항
**고객 관점**
- 가격 투명성이 최우선 니즈
- 진행상황 실시간 확인 강력 요구
- 3단계 이내 간단한 예약 프로세스 필요
- 리뷰보다는 지인 추천을 더 신뢰

**정비소 관점**  
- 예약 관리 자동화가 최우선 필요
- 수수료에 대한 민감도 높음
- 기술 학습에 대한 부담 존재
- 신규 고객 확보에 대한 강한 니즈

### 4.2 경쟁사 대비 차별화 포인트
- **투명한 가격 정보**: 세부 항목별 가격 공개
- **실시간 진행 추적**: 사진과 함께 단계별 업데이트
- **간편한 예약 프로세스**: 3단계 완료
- **정비소 친화적 인터페이스**: 쉬운 관리 도구

### 4.3 기능 우선순위 재조정
**P0 (MVP) 기능 확정**
1. 간편 예약 시스템
2. 투명한 견적 시스템  
3. 기본 결제 기능
4. 정비소 관리 대시보드

**P1 기능으로 조정**
- 실시간 진행상황 추적
- 정비 이력 관리
- 리뷰 시스템

**P2 기능으로 연기**
- AI 추천 시스템
- 고급 분석 기능

## 5. 설계 변경 사항

### 5.1 UI/UX 개선 사항
**예약 프로세스 단순화**
- 4단계 → 3단계로 줄임
- 차량 정보 입력 간소화
- 필수/선택 항목 명확히 구분

**가격 정보 강화**
- 메인 화면에 가격 범위 표시
- 세부 견적서 실시간 계산
- 추가 비용 가능성 사전 안내

**진행상황 시각화**
- 6단계 프로세스 진행바
- 각 단계별 예상 시간 표시
- 사진 첨부 기능 강화

### 5.2 비즈니스 모델 조정
**수수료 정책**
- 기본 수수료: 8% (기존 10%에서 하향)
- 초기 3개월 할인: 5%
- 거래량 기반 차등 적용

**정비소 지원 정책**
- 무료 교육 프로그램 제공
- 전담 CS 지원
- 마케팅 도구 무료 제공
