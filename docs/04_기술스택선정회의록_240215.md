**참석자**: 이기술(CTO), 개발자 A(백엔드리드), 개발자 B(프론트엔드리드), 개발자 C, 외부 아키텍트 컨설턴트

## 1. 기술 스택 선정 기준

### 1.1 선정 기준 정의
- **개발 생산성**: 빠른 MVP 출시 (4개월 목표)
- **확장성**: 향후 사용자 증가 대응
- **유지보수성**: 팀 규모 확장 고려
- **비용 효율성**: 초기 스타트업 예산 고려
- **인력 수급**: 개발자 채용 용이성
- **안정성**: 검증된 기술 우선

### 1.2 비교 대상 기술들
**프론트엔드**: React.js vs Vue.js vs Angular
**백엔드**: Node.js vs Java Spring vs Python Django
**데이터베이스**: PostgreSQL vs MySQL vs MongoDB
**클라우드**: AWS vs Azure vs GCP

## 2. 프론트엔드 기술 스택 선정

### 2.1 후보 기술 비교

| 기준 | React.js | Vue.js | Angular |
|------|----------|--------|---------|
| 학습 곡선 | 보통 | 쉬움 | 어려움 |
| 생태계 | 매우 풍부 | 보통 | 풍부 |
| 성능 | 우수 | 우수 | 우수 |
| 인력 확보 | 용이 | 보통 | 어려움 |
| 커뮤니티 | 매우 활발 | 활발 | 활발 |

### 2.2 최종 선정: **React.js**
**선정 이유**
- 풍부한 생태계와 라이브러리
- 팀 구성원 경험 보유
- 모바일 대응 (React Native 확장 가능)
- 대기업 백업 (Meta)

**함께 사용할 기술**
- **TypeScript**: 코드 안정성 향상
- **Tailwind CSS**: 빠른 UI 개발
- **React Query**: 서버 상태 관리
- **PWA**: 네이티브 앱 수준 경험

## 3. 백엔드 기술 스택 선정

### 3.1 후보 기술 비교

| 기준 | Java Spring | Node.js | Python Django |
|------|-------------|---------|---------------|
| 성능 | 우수 | 좋음 | 보통 |
| 확장성 | 매우 우수 | 좋음 | 좋음 |
| 개발 속도 | 보통 | 빠름 | 빠름 |
| 엔터프라이즈 | 매우 적합 | 보통 | 보통 |
| 인력 확보 | 용이 | 용이 | 보통 |
| 트랜잭션 처리 | 우수 | 보통 | 좋음 |

### 3.2 최종 선정: **Java SpringBoot**
**선정 이유**
- 결제 시스템 등 금융 거래에 안정적
- 마이크로서비스 아키텍처 지원 우수
- 대용량 트래픽 처리 경험 풍부
- 기업용 솔루션으로 검증됨
- ORM(MyBatis) 및 트랜잭션 관리 용이

**함께 사용할 기술**
- **Spring Security**: 인증/권한 관리
- **MyBatis**: 데이터베이스 매핑
- **Redis**: 캐싱 및 세션 관리
- **JWT**: 토큰 기반 인증

## 4. 데이터베이스 선정

### 4.1 후보 기술 비교

| 기준 | PostgreSQL | MySQL | MongoDB |
|------|------------|-------|---------|
| 성능 | 우수 | 우수 | 좋음 |
| 확장성 | 우수 | 좋음 | 매우 우수 |
| 안정성 | 매우 우수 | 우수 | 좋음 |
| 복잡 쿼리 | 매우 우수 | 우수 | 보통 |
| JSON 지원 | 우수 | 보통 | 매우 우수 |
| 오픈소스 | 완전 무료 | 상업적 제약 | 무료(제한) |

### 4.2 최종 선정: **PostgreSQL**
**선정 이유**
- 복잡한 비즈니스 로직 처리 우수
- JSONB 지원으로 유연한 데이터 구조
- 완전한 오픈소스 (라이센스 이슈 없음)
- 지리 정보 처리 (PostGIS) 우수
- ACID 속성 완벽 지원

**보조 데이터베이스**
- **Redis**: 캐싱, 세션, 실시간 데이터
- **AWS S3**: 파일 저장소 (차량 사진 등)

## 5. 클라우드 및 인프라

### 5.1 클라우드 선정: **AWS**
**선정 이유**
- 가장 성숙한 클라우드 생태계
- 다양한 관리형 서비스
- 한국 리전 제공으로 지연시간 최소화
- 상세한 한국어 문서 및 지원

**주요 사용 서비스**
- **EC2**: 애플리케이션 서버
- **RDS**: PostgreSQL 관리형 서비스
- **ElastiCache**: Redis 관리형 서비스
- **S3**: 파일 저장소
- **CloudFront**: CDN
- **Route 53**: DNS 관리

### 5.2 컨테이너화: **Docker**
- 개발/운영 환경 일치
- 마이크로서비스 배포 용이
- CI/CD 파이프라인 구축

## 6. 외부 API 및 서비스

### 6.1 결제 시스템
**주 결제사**: 토스페이먼츠
- 개발자 친화적 API
- 다양한 결제 수단 지원
- 합리적인 수수료

**보조 결제사**: 이니시스
- 전통적 PG사, 안정성 우수
- 기업 결제 지원

### 6.2 지도 서비스: **카카오맵 API**
- 국내 지도 정확도 우수
- 검색 기능 강력
- 개발 문서 충실

### 6.3 SMS 발송: **알리고**
- 비용 효율적
- 대량 발송 지원
- 발송 성공률 높음

### 6.4 간편결제 연동
- **카카오페이**: 사용자 수 최다
- **네이버페이**: 안정성 우수  
- **토스페이**: 젊은 층 선호
