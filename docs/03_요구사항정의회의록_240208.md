## 1. 사용자 유형 및 권한 정의

### 1.1 주요 사용자 그룹
**일반 고객 (Customer)**
- 차량 소유자, 정비 서비스 이용
- 나이: 25-60세, 스마트폰 활용 가능
- 핵심 니즈: 편리한 예약, 투명한 가격, 신뢰할 수 있는 정비소

**정비소 직원 (Service Center Staff)**
- 정비사, 상담원, 관리자
- 기존 시스템 사용 경험 제한적
- 핵심 니즈: 간편한 예약 관리, 고객 소통, 매출 증대

**시스템 관리자 (System Admin)**
- 플랫폼 운영진
- 핵심 니즈: 전체 현황 모니터링, 분쟁 처리, 정산 관리

### 1.2 사용자별 주요 기능

**고객 기능**
- [ ] 회원가입/로그인 (이메일, 소셜로그인)
- [ ] 차량 정보 등록 및 관리
- [ ] 정비소 검색 (위치, 평점, 서비스 기준)
- [ ] 예약 생성/수정/취소
- [ ] 실시간 진행상황 확인
- [ ] 결제 및 영수증 관리
- [ ] 정비 이력 조회
- [ ] 리뷰 작성

**정비소 직원 기능**
- [ ] 예약 현황 대시보드
- [ ] 차량 접수 및 상태 업데이트
- [ ] 견적서 생성 및 발송
- [ ] 정비 진행상황 사진/코멘트 등록
- [ ] 고객과의 메시지 소통
- [ ] 매출 및 통계 조회

## 2. 핵심 비즈니스 프로세스

### 2.1 예약 프로세스
**Step 1: 정비소 선택**
- 위치 기반 검색 (반경 30km 이내)
- 필터링: 서비스 종류, 평점, 가격대
- 정비소 상세 정보: 운영시간, 리뷰, 제공 서비스

**Step 2: 서비스 선택**
- 정비 항목 카테고리 분류:
  - 엔진 관련 (오일교체, 필터교체 등)
  - 브레이크 시스템
  - 타이어 관련
  - 배터리/전기 시스템
  - 정기점검
- 다중 선택 가능
- 각 항목별 예상 가격 및 소요시간 표시

**Step 3: 날짜/시간 선택**
- 캘린더 형태 UI
- 실시간 예약 가능 시간 표시
- 예상 소요시간 고려한 슬롯 계산

**Step 4: 차량 정보 입력**
- 기존 등록 차량 선택 또는 신규 등록
- 필수 정보: 차량번호, 제조사, 모델, 연식
- 추가 정보: 주행거리, 특이사항

**Step 5: 예약 확정**
- 최종 견적 확인
- 예약금 결제 (전체 금액의 10-30%)
- 확인 SMS/이메일 발송

### 2.2 정비 진행 프로세스
**6단계 진행 상황 관리**
1. **접수 완료**: 차량 입고, QR코드 스캔
2. **진단 중**: 차량 상태 점검, 문제 파악
3. **견적 대기**: 상세 견적서 발송, 고객 승인 대기
4. **정비 진행**: 실제 수리 작업 수행
5. **품질 검사**: 작업 완료 후 최종 점검
6. **완료**: 차량 인도 준비, 결제 완료

**각 단계별 요구사항**
- 실시간 상태 업데이트
- 단계별 사진 첨부 (선택사항)
- 정비사 코멘트 작성
- 예상 완료 시간 계산 및 업데이트
- 추가 정비 발견 시 즉시 고객 알림

## 3. 기능별 상세 요구사항

### 3.1 예약 시스템
**기본 기능**
- 실시간 예약 가능 시간 확인
- 예약 충돌 방지 로직
- 예약 변경/취소 기능 (규정 내)
- 예약 리마인더 알림

**고급 기능 (Phase 2)**
- 대기 예약 기능
- 단골 고객 우대 예약
- 그룹 예약 (여러 차량 동시)

### 3.2 결제 시스템
**지원 결제 수단**
- 신용카드/체크카드
- 계좌이체
- 휴대폰 결제
- 카카오페이/네이버페이/토스페이

**결제 타이밍**
- 예약 시 부분 결제 (예약금)
- 작업 완료 후 잔금 결제
- 고액 정비 시 분할 결제 옵션

### 3.3 알림 시스템
**알림 채널**
- SMS (기본)
- 이메일
- 푸시 알림 (PWA)

**알림 타이밍**
- 예약 확정/변경/취소 시
- 정비 단계 변경 시
- 추가 정비 필요 시
- 차량 인도 가능 시
- 정기점검 시기 도래 시

## 4. 비기능 요구사항

### 4.1 성능 요구사항
- 페이지 로딩 시간: 3초 이내
- 동시 접속자: 100명 지원
- 시스템 가용성: 99% 이상
- 모바일 반응속도: 1초 이내

### 4.2 보안 요구사항
- HTTPS 통신 필수
- 개인정보 암호화 저장
- 결제 정보 토큰화
- 정기적 보안 감사

### 4.3 사용성 요구사항
- 모바일 우선 설계
- 3단계 이내 주요 작업 완료
- 직관적인 아이콘 및 메뉴
- 접근성 표준 준수

## 5. 데이터 요구사항

### 5.1 핵심 데이터 엔티티
- **사용자**: 고객/직원 정보, 권한
- **차량**: 차종, 연식, 정비 이력
- **정비소**: 기본 정보, 서비스, 운영시간
- **예약**: 예약 정보, 상태, 이력
- **견적**: 서비스별 가격, 할인 정보
- **결제**: 거래 내역, 정산 정보

### 5.2 데이터 보관 정책
- 개인정보: 탈퇴 후 즉시 삭제
- 거래 기록: 7년간 보관 (세법 준수)
- 로그 데이터: 1년간 보관

## 6. 외부 연동 요구사항

### 6.1 필수 연동
- **결제 게이트웨이**: 이니시스, KG이니시스
- **지도 서비스**: 카카오맵 API
- **SMS 서비스**: 알리고 또는 네이버 클라우드
- **간편결제**: 카카오페이, 네이버페이, 토스

### 6.2 선택적 연동
- **차량 정보**: 교통안전공단 API
- **소셜 로그인**: Google, Naver, Kakao
- **이메일**: SendGrid 또는 AWS SES
