# 자동차 정비 예약 플랫폼 프로젝트 킥오프 회의록

**회의명**: CarFix 플랫폼 개발 프로젝트 킥오프  
**일시**: 2024년 2월 1일 14:00-16:00  
**장소**: 본사 대회의실  
**참석자**: 김사업(대표), 이기술(CTO), 박영업(영업이사), 정디자인(디자인팀장), 개발자 A, B, C

## 1. 프로젝트 개요 확인

### 1.1 비전 및 목표 합의
- **비전**: "국내 1위 자동차 정비 O2O 플랫폼"
- **1년 목표**: 지역 시장 점유율 10% 달성
- **핵심 가치**: 투명성, 편의성, 신뢰성

### 1.2 성공 지표 정의
- 월간 활성 사용자(MAU): 10,000명
- 입점 정비소: 100개소
- 월 거래액: 1억원

## 2. 프로젝트 스코프 및 일정

### 2.1 개발 범위
**Phase 1 (MVP) - 4개월**
- 기본 사용자 관리 (회원가입/로그인)
- 정비소 검색 및 예약
- 결제 시스템
- 관리자 대시보드

**Phase 2 - 추가 2개월**
- 실시간 진행상황 추적
- 리뷰 및 평점 시스템
- 푸시 알림

**Phase 3 - 향후 확장**
- AI 기반 추천
- 고급 분석 기능

### 2.2 주요 마일스톤
- **2월 말**: 요구사항 정의 완료
- **3월 말**: 프로토타입 완성
- **4월 말**: Alpha 버전
- **6월 말**: Beta 출시
- **8월 말**: 정식 출시

## 3. 팀 구성 및 역할

### 3.1 개발팀
- **이기술 (CTO)**: 전체 기술 책임, 아키텍처 설계
- **개발자 A**: 백엔드 리드
- **개발자 B**: 프론트엔드 리드  
- **개발자 C**: 풀스택 개발자

### 3.2 지원팀
- **박영업 (영업이사)**: 정비소 파트너십
- **정디자인 (디자인팀장)**: UI/UX 설계
- **김사업 (대표)**: 전체 기획 및 의사결정

## 4. 주요 결정사항

### 4.1 기술 방향성
- **웹 우선 개발**: 빠른 출시를 위해 반응형 웹으로 시작
- **클라우드 네이티브**: AWS 기반 마이크로서비스 아키텍처
- **외부 서비스 적극 활용**: 결제(PG사), 지도(네이버/카카오), 인증(소셜로그인)

### 4.2 사용자 경험 우선순위
1. **간편한 예약 프로세스**: 3단계 이내 예약 완료
2. **투명한 가격 표시**: 사전 견적 필수
3. **실시간 소통**: 정비 진행상황 실시간 확인

### 4.3 비즈니스 모델
- **중개 수수료**: 거래액의 10% (협상 가능)
- **프리미엄 서비스**: 정비소 상위 노출
- **향후 확장**: 부품 판매, 보험 연계
