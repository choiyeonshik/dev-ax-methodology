# 자동차 정비 예약 플랫폼 요구사항 정의서

**문서명**: 요구사항 정의서 (Requirements Specification)  
**프로젝트**: 자동차 정비 예약 플랫폼 (CarFix)  
**버전**: v2.0  
**작성일**: 2024년 2월 28일  
**승인일**: 2024년 3월 5일  
**작성자**: 개발팀, 기획팀  
**승인자**: 김사업 대표, 이기술 CTO

## 1. 문서 개요

### 1.1 목적
본 문서는 자동차 정비 예약 플랫폼의 기능적 요구사항과 비기능적 요구사항을 명확히 정의하여, 개발팀과 이해관계자 간의 공통된 이해를 제공함을 목적으로 합니다.

### 1.2 범위
- 웹 기반 자동차 정비 예약 플랫폼
- 고객용 서비스, 정비소용 관리 시스템, 시스템 관리자용 대시보드
- 외부 시스템 연동 (결제, 지도, 알림 등)

### 1.3 대상 독자
- 개발팀 (프론트엔드, 백엔드, DevOps)
- 기획팀 및 사업팀
- QA 및 테스트팀
- 외부 파트너 (정비소, 외부 개발사)

## 2. 시스템 개요

### 2.1 시스템 목적
자동차 정비가 필요한 고객과 정비소를 온라인으로 연결하여, 투명하고 편리한 정비 예약 서비스를 제공합니다.

### 2.2 주요 이해관계자

| 이해관계자 | 역할 | 주요 니즈 |
|------------|------|----------|
| **고객** | 차량 소유자, 정비 서비스 이용자 | 편리한 예약, 투명한 가격, 진행상황 확인 |
| **정비소 직원** | 예약 관리, 정비 작업 수행 | 효율적 예약 관리, 고객 소통, 매출 증대 |
| **시스템 관리자** | 플랫폼 운영 및 관리 | 전체 현황 모니터링, 분쟁 해결, 수익 관리 |

### 2.3 주요 성공 기준
- 월간 활성 사용자(MAU): 10,000명
- 입점 정비소: 100개소  
- 예약 완료율: 85% 이상
- 고객 만족도: 4.5점/5점 이상

## 3. 기능적 요구사항

### 3.1 사용자 관리 (USER)

#### 3.1.1 회원가입 및 인증
**REQ-USER-001: 이메일 회원가입**
- **설명**: 사용자는 이메일 주소를 통해 회원가입할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 이메일 주소 유효성 검증
  - 이메일 인증을 통한 계정 활성화
  - 비밀번호 8자리 이상, 영문/숫자/특수문자 조합
  - 이용약관 및 개인정보처리방침 동의 필수
- **승인 조건**:
  - [ ] 유효하지 않은 이메일 형식 입력 시 오류 메시지 표시
  - [ ] 중복 이메일 가입 시도 시 적절한 안내 메시지
  - [ ] 이메일 인증 후 로그인 가능
  - [ ] 약관 미동의 시 회원가입 불가

**REQ-USER-002: 소셜 로그인**
- **설명**: 구글, 네이버, 카카오 계정으로 간편 로그인 가능
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - Google OAuth 2.0 연동
  - 네이버 아이디로 로그인 API 연동
  - 카카오 로그인 API 연동
  - 소셜 계정 정보 자동 매핑
- **승인 조건**:
  - [ ] 각 소셜 플랫폼에서 3초 이내 로그인 완료
  - [ ] 소셜 계정 정보 정확히 가져오기
  - [ ] 기존 이메일 계정과 연동 가능

**REQ-USER-003: 프로필 관리**
- **설명**: 사용자는 개인 정보를 수정하고 관리할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 이름, 전화번호, 주소 수정
  - 비밀번호 변경 (기존 비밀번호 확인 필수)
  - 알림 설정 (SMS, 이메일, 푸시)
  - 계정 삭제 (회원 탈퇴)
- **승인 조건**:
  - [ ] 개인정보 수정 후 즉시 반영
  - [ ] 비밀번호 변경 시 보안 규칙 적용
  - [ ] 계정 삭제 시 개인정보 완전 삭제

#### 3.1.2 정비소 직원 관리
**REQ-USER-004: 정비소 직원 등록**
- **설명**: 정비소 관리자는 직원을 등록하고 권한을 부여할 수 있어야 함
- **우선순위**: P1 (중요)
- **상세 요구사항**:
  - 직원 기본 정보 등록 (이름, 연락처, 자격증)
  - 역할별 권한 설정 (관리자, 정비사, 상담원)
  - 근무 스케줄 설정
  - 전문 분야 설정 (엔진, 브레이크, 전기 등)

### 3.2 차량 정보 관리 (VEHICLE)

**REQ-VEHICLE-001: 차량 등록**
- **설명**: 고객은 소유 차량 정보를 등록하고 관리할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 차량번호 입력 및 유효성 검증
  - 제조사, 모델, 연식 선택
  - 엔진 타입 선택 (가솔린, 디젤, 하이브리드, 전기)
  - 현재 주행거리 입력
  - 차량 사진 등록 (선택사항)
  - 여러 대 차량 등록 지원
- **승인 조건**:
  - [ ] 차량번호 중복 등록 방지
  - [ ] 제조사/모델 데이터베이스 연동
  - [ ] 차량 사진 3MB 이내 업로드 지원
  - [ ] 최대 5대까지 등록 가능

**REQ-VEHICLE-002: 차량 정비 이력 관리**
- **설명**: 시스템은 차량별 정비 이력을 자동으로 기록하고 관리해야 함
- **우선순위**: P1 (중요)
- **상세 요구사항**:
  - 정비 완료 시 자동 이력 저장
  - 정비 항목, 비용, 교체 부품 기록
  - 다음 정비 권장 시기 계산
  - 정비 이력 검색 및 필터링
  - 정비 이력 내보내기 (PDF, Excel)

### 3.3 정비소 관리 (SERVICE_CENTER)

**REQ-SC-001: 정비소 정보 관리**
- **설명**: 정비소는 기본 정보와 서비스 정보를 등록하고 관리할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 정비소 기본 정보 (상호명, 사업자번호, 주소, 연락처)
  - 운영시간 설정 (평일/주말/공휴일 별도)
  - 제공 서비스 목록 및 가격 설정
  - 정비소 사진 및 소개글 등록
  - 위치 정보 (지도 연동)
- **승인 조건**:
  - [ ] 사업자번호 유효성 검증
  - [ ] 카카오맵 API 연동한 정확한 위치 표시
  - [ ] 운영시간 외 예약 요청 시 안내 메시지

**REQ-SC-002: 서비스 및 가격 관리**
- **설명**: 정비소는 제공 서비스와 가격을 관리할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 서비스 카테고리별 분류 (엔진, 브레이크, 타이어 등)
  - 서비스별 기본 가격 설정
  - 차종별 추가 요금 설정
  - 프로모션 및 할인 설정
  - 서비스 일시 중단 설정

### 3.4 예약 시스템 (BOOKING)

**REQ-BOOKING-001: 정비소 검색**
- **설명**: 고객은 조건에 맞는 정비소를 검색할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 현재 위치 기반 반경 검색 (5km, 10km, 20km)
  - 서비스 종류별 필터링
  - 평점 및 리뷰 기준 정렬
  - 가격대별 필터링
  - 운영 상태 (영업중/휴무) 표시
- **승인 조건**:
  - [ ] 검색 결과 3초 이내 표시
  - [ ] 위치 기반 정확한 거리 계산
  - [ ] 실시간 운영 상태 반영

**REQ-BOOKING-002: 예약 생성**
- **설명**: 고객은 원하는 정비소에 예약을 생성할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 3단계 예약 프로세스 (정비소 선택 → 서비스 선택 → 일시 선택)
  - 실시간 예약 가능 시간 확인
  - 여러 서비스 동시 선택 가능
  - 차량 정보 자동 연결
  - 특별 요청사항 입력
  - 예상 견적 실시간 계산
- **승인 조건**:
  - [ ] 예약 프로세스 3단계 이내 완료
  - [ ] 예약 중복 방지 로직 정상 작동
  - [ ] 예약 확정 후 1분 이내 확인 알림 발송

**REQ-BOOKING-003: 예약 관리**
- **설명**: 고객은 생성한 예약을 조회, 수정, 취소할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 예약 목록 조회 (예정/진행중/완료/취소)
  - 예약 상세 정보 확인
  - 예약 일시 변경 (24시간 전까지)
  - 예약 취소 (취소 정책 적용)
  - 예약 상태별 필터링
- **승인 조건**:
  - [ ] 예약 수정 시 실시간 가능 시간 확인
  - [ ] 취소 정책에 따른 취소 수수료 자동 계산
  - [ ] 취소 시 즉시 다른 대기 고객에게 알림

### 3.5 견적 및 결제 시스템 (PAYMENT)

**REQ-PAYMENT-001: 사전 견적 시스템**
- **설명**: 고객은 예약 전에 예상 견적을 확인할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 서비스별 기본 가격 표시
  - 차종에 따른 추가 요금 계산
  - 부가세 포함/제외 가격 선택
  - 할인 쿠폰 적용 가능 여부 표시
  - 견적서 저장 및 공유 기능
- **승인 조건**:
  - [ ] 견적 계산 1초 이내 완료
  - [ ] 실제 결제 금액과 견적 오차 5% 이내
  - [ ] 견적서 PDF 다운로드 가능

**REQ-PAYMENT-002: 결제 처리**
- **설명**: 시스템은 다양한 결제 수단을 지원하고 안전하게 처리해야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 신용카드/체크카드 결제
  - 계좌이체 결제
  - 간편결제 (카카오페이, 네이버페이, 토스페이)
  - 휴대폰 결제
  - 예약금 부분 결제 지원
  - PG사 연동 (토스페이먼츠, 이니시스)
- **승인 조건**:
  - [ ] 결제 처리 3초 이내 완료
  - [ ] 결제 실패 시 명확한 오류 메시지
  - [ ] 결제 완료 후 즉시 영수증 발송
  - [ ] PCI DSS 보안 기준 준수

**REQ-PAYMENT-003: 환불 처리**
- **설명**: 예약 취소 또는 서비스 문제 시 환불을 처리할 수 있어야 함
- **우선순위**: P1 (중요)
- **상세 요구사항**:
  - 취소 정책에 따른 자동 환불 계산
  - 부분 환불 지원
  - 환불 승인 워크플로우
  - 환불 처리 상태 추적
  - 환불 완료 알림

### 3.6 정비 진행상황 추적 (PROGRESS)

**REQ-PROGRESS-001: 실시간 진행상황 추적**
- **설명**: 고객은 정비 진행상황을 실시간으로 확인할 수 있어야 함
- **우선순위**: P1 (중요)
- **상세 요구사항**:
  - 6단계 진행상황 표시 (접수→진단→견적→정비→검사→완료)
  - 각 단계별 예상 완료 시간 표시
  - 진행바 형태의 시각적 표현
  - 단계별 상세 설명 및 사진 첨부
  - 실시간 업데이트 (WebSocket 또는 SSE)
- **승인 조건**:
  - [ ] 상태 변경 시 5초 이내 고객 화면 반영
  - [ ] 각 단계별 사진 최소 1장 이상 첨부
  - [ ] 예상 완료 시간 정확도 80% 이상

**REQ-PROGRESS-002: 정비사 진행상황 업데이트**
- **설명**: 정비사는 작업 진행상황을 쉽게 업데이트할 수 있어야 함
- **우선순위**: P1 (중요)
- **상세 요구사항**:
  - 모바일 친화적 진행상황 업데이트 인터페이스
  - 사진 촬영 및 즉시 업로드
  - 음성 메모 녹음 및 텍스트 변환
  - 표준 코멘트 템플릿 제공
  - 추가 정비 필요 시 즉시 고객 알림

### 3.7 알림 시스템 (NOTIFICATION)

**REQ-NOTI-001: 다채널 알림 발송**
- **설명**: 시스템은 다양한 채널을 통해 알림을 발송할 수 있어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - SMS 알림 (기본)
  - 이메일 알림
  - 웹 푸시 알림 (PWA)
  - 인앱 알림
  - 사용자별 알림 채널 선택 가능
- **승인 조건**:
  - [ ] SMS 발송 성공률 95% 이상
  - [ ] 이메일 발송 3초 이내 완료
  - [ ] 푸시 알림 즉시 전송

**REQ-NOTI-002: 알림 트리거 조건**
- **설명**: 특정 이벤트 발생 시 자동으로 알림이 발송되어야 함
- **우선순위**: P0 (필수)
- **상세 요구사항**:
  - 예약 확정/변경/취소 시 즉시 알림
  - 정비 단계 변경 시 즉시 알림
  - 추가 정비 필요 시 즉시 알림
