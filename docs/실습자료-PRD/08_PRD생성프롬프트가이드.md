# PRD 템플릿 기반 완전한 PRD 생성 가이드

**문서명**: PRD_Template.txt를 활용한 완전한 PRD 생성 가이드  
**버전**: v2.0  
**작성일**: 2024년 3월 8일  
**작성자**: @TechPM (Orchestra AI)  
**목적**: `docs/PRD_Template.txt` 템플릿을 활용하여 Cursor AI 최적화된 PRD 생성

## 1. 가이드 개요

### 1.1 새로운 PRD 생성 방법론
이 가이드는 **PRD_Template.txt 템플릿**을 기반으로 하여 체계적이고 완전한 PRD를 생성하는 혁신적인 방법을 제시합니다.

**핵심 혁신 요소**
- ✅ **템플릿 기반 접근**: [대괄호] 방식의 구조화된 작성
- ✅ **Cursor AI 최적화**: Task Master AI 연동 준비된 형식
- ✅ **체크리스트 승인 조건**: 명확한 완료 기준
- ✅ **우선순위 기반 개발**: P0/P1/P2 명확한 구분

**주요 활용 시나리오**
- 신규 디지털 서비스/플랫폼 기획
- 기존 서비스의 기능 확장 PRD 작성
- 스타트업의 MVP 정의 및 기획  
- 대기업의 신사업 기획서 작성
- AI 기반 자동 태스크 생성 시스템 구축

### 1.2 템플릿 기반 PRD 생성 프로세스
```
입력자료 수집 → 템플릿 복사 → [대괄호] 항목 분석 → 
섹션별 작성 → 승인 조건 정의 → 품질 검증 → 최종 PRD 완성
```

## 2. PRD 템플릿 활용 준비

### 2.1 템플릿 기본 구조 이해
`docs/PRD_Template.txt`는 **[대괄호] 방식**으로 구성된 완전한 PRD 템플릿입니다.

**핵심 구조 (10개 섹션)**
```
1. 프로젝트 개요 → 비즈니스 목표 + 사용자 시나리오
2. 기능 요구사항 → 13개 카테고리 + P0/P1/P2 우선순위  
3. 비기능 요구사항 → 성능/보안/사용성
4. 시스템 아키텍처 및 기술스택
5. 데이터 모델 → 핵심 엔티티 정의
6. ⭐ 주요 기능별 승인 조건 → 체크리스트 형태
7. 테스트 전략 → 단위/통합/성능/보안
8. 개발 일정 및 마일스톤
9. 품질 보증 및 유지보수
10. 결론 및 사용 방법
```

### 2.2 입력자료 매핑 전략

**[대괄호] → 입력자료 매핑 예시**
- `[프로젝트명]` ← 사업 기획서의 서비스명
- `[Primary User]` ← 사용자 인터뷰 결과의 주 사용자군
- `[기간]` ← 프로젝트 계획서의 개발 일정
- `[목표 수치]` ← 비즈니스 목표의 정량적 지표
- `[기술스택]` ← 기술 선정 문서의 확정된 스택

### 2.3 필수 입력자료 (Must Have)
- [ ] **사업 기획서**: [프로젝트명], [비즈니스 목표] 정보원
- [ ] **사용자 인터뷰**: [Primary User], [Secondary User] 정의원
- [ ] **기술 선정 문서**: [기술스택], [아키텍처] 명세원
- [ ] **요구사항 정의**: [기능 카테고리], [우선순위] 분류원

### 2.4 권장 입력자료 (Nice to Have)  
- [ ] **아키텍처 설계서**: [데이터 모델], [외부 연동] 상세원
- [ ] **프로젝트 일정**: [마일스톤], [위험 관리] 계획원
- [ ] **품질 기준**: [승인 조건], [테스트 전략] 기준원

## 3. 템플릿 기반 PRD 작성 단계

### 3.1 Step 1: 템플릿 준비 및 초기 분석

**🔥 핵심 프롬프트**
```
@Tech PM 역할로 다음 과정을 수행해주세요:

1. docs/PRD_Template.txt를 복사하여 새 PRD 파일 생성
2. 다음 입력자료를 분석하여 템플릿의 [대괄호] 항목 매핑

입력자료:
- @[사업기획서파일명]
- @[사용자인터뷰결과파일명]  
- @[기술선정문서파일명]
- @[요구사항정의파일명]

템플릿 매핑 우선순위:
1. [프로젝트명] → 서비스/제품명 확정
2. [Primary User] → 주요 사용자 그룹 정의
3. [기술스택] → 확정된 기술 스택 반영
4. [목표 수치] → 비즈니스 KPI 수치 설정
5. [기간] → 개발 마일스톤 일정 반영

출력: [대괄호] 항목별 매핑 결과 테이블
```

**기대 결과**
- 템플릿 기반 PRD 파일 생성 (`[프로젝트명]_PRD.txt`)
- 핵심 [대괄호] 항목들의 구체적 정의
- 입력자료 기반 정보 매핑 완료

### 3.2 Step 2: 섹션별 [대괄호] 항목 작성

**🎯 섹션별 작성 프롬프트**
```
PRD 템플릿의 "[섹션명]" 부분을 다음 단계로 작성해주세요:

1단계: [대괄호] 항목 식별
- 해당 섹션의 모든 [대괄호] 항목 목록 추출
- 각 항목별 작성 우선순위 결정

2단계: 입력자료 기반 내용 작성
참조 자료: @[관련 입력자료들]
작성 원칙:
- 구체적이고 측정 가능한 표현 사용
- 모호한 [대괄호]는 "구체적 값 + 단위" 형태로 대체
- 우선순위(P0/P1/P2) 명확히 구분

3단계: 품질 검증
- 다른 섹션과의 일관성 확인
- 실행 가능성 검토
- 측정 가능성 검토

예시 변환:
[시간] → "3초" 
[숫자] → "100명"
[기능 카테고리 1] → "사용자 관리"
```

### 3.3 Step 3: 승인 조건 체크리스트 완성

**📋 핵심 프롬프트**
```
PRD 템플릿의 "6. 주요 기능별 승인 조건" 섹션을 다음 형태로 완성해주세요:

기능별 승인 조건 작성 규칙:
1. 측정 가능한 기준만 포함
2. 성공/실패 판단이 명확해야 함
3. 시간 기준 포함 (응답시간, 처리시간 등)
4. 정량적 수치 포함 (성공률, 정확도 등)

예시 변환:
❌ [구체적 조건 1] → ✅ "사용자 로그인 3초 이내 완료"
❌ [구체적 조건 2] → ✅ "예약 성공률 95% 이상 달성"  
❌ [구체적 조건 3] → ✅ "결제 실패 시 명확한 오류 메시지 표시"

각 기능당 5개 승인 조건 필수 작성:
- 성능 기준 (시간, 처리량)
- 품질 기준 (성공률, 정확도)
- 사용성 기준 (단계수, 오류 처리)
- 보안 기준 (인증, 데이터 보호)  
- 호환성 기준 (브라우저, 디바이스)
```

### 3.4 Step 4: Task Master AI 최적화

**🤖 AI 연동 최적화 프롬프트**
```
완성된 PRD를 Task Master AI 연동에 최적화해주세요:

1단계: 우선순위 기반 태스크 구조 검증
- P0 기능들이 명확한 태스크 단위로 분리되었는지 확인
- 각 태스크가 독립적으로 수행 가능한지 검증
- 의존성 관계가 명확하게 정의되었는지 확인

2단계: 승인 조건 체크리스트 최적화  
- 모든 승인 조건이 [ ] 체크박스 형태인지 확인
- 자동 검증 가능한 조건과 수동 검증 조건 구분
- Task Master가 이해할 수 있는 명확한 언어 사용

3단계: 개발 일정 연동 준비
- 각 마일스톤이 측정 가능한 산출물로 정의되었는지 확인
- 위험 관리 계획이 구체적인 대응 방안 포함하는지 검증

출력: Task Master AI 연동 준비도 점검표
```

### 4.3 우선순위별 기능 정의 템플릿

**🎯 P0/P1/P2 분류 프롬프트**
```
PRD 템플릿의 "2.13 기능별 개발 우선순위" 섹션을 완성해주세요:

참조 자료: @[요구사항정의서] @[사용자인터뷰결과]

분류 기준:
**P0 (MVP 필수)**: 서비스 기본 작동에 절대 필요
- 사용자가 핵심 가치를 경험할 수 있는 최소 기능
- 이것 없으면 서비스 출시 불가능

**P1 (초기 확장)**: 사용자 만족도 및 경쟁력 향상
- P0 완료 후 3개월 내 추가 필요
- 사용자 retention 및 경험 개선

**P2 (장기 고도화)**: 차별화 및 시장 확장
- 6-12개월 후 시장 확장용
- AI/고급 기능/새로운 사용자군 타겟

작성 형태:
#### P0 (필수 - MVP 기능)
- [기능 1] → "사용자 인증" (소셜로그인 포함)
- [기능 2] → "기본 예약" (날짜/시간 선택)
...

각 기능에 선정 근거 1문장 추가
```

## 5. 템플릿 품질 검증 체계

### 5.1 [대괄호] 완성도 체크리스트
