# CarFix 자동차 정비 예약 플랫폼 완전한 PRD
# 본 문서는 Cursor AI와 Task Master AI 구현 기준 문서입니다.

## 1. 프로젝트 개요

### 1.1 시스템 목적
- 정비 예약 과정을 온라인화하여 예약 접근성과 처리 효율을 높인다.
- 사전 견적과 단계별 진행 공유로 가격/과정 투명성을 강화한다.
- 고객, 정비소, 운영자가 동일한 데이터 기반으로 협업하는 운영 체계를 구축한다.

### 1.2 주요 사용자
- **Primary User (차량 소유자)**: 25-60세, 모바일 중심 사용, 빠른 예약/투명한 가격/신뢰 가능한 정비소를 원함.
- **Secondary User (정비소 관리자/직원)**: 디지털 도구 숙련도 편차가 크며, 예약 자동화/고객 소통/매출 관리 니즈가 큼.
- **Internal User (플랫폼 운영 관리자)**: 전체 운영 지표 모니터링, 분쟁 처리, 정산 및 정책 관리 담당.

### 1.3 비즈니스 목표 및 성공 기준

#### 1.3.1 주요 비즈니스 목표
- **시장 점유 목표**: 서비스 출시 1년 내 지역 시장 점유율 10% 달성.
- **사용자 목표**: 출시 12개월 내 MAU 10,000명 달성.
- **파트너 목표**: 출시 12개월 내 입점 정비소 100개소 달성.
- **거래 목표**: 출시 12개월 내 월 거래액 1억원 달성.

#### 1.3.2 핵심 성과 지표(KPI)
- **사용자 지표**
  - 예약 완료율: 85% 이상
  - 고객 만족도: 4.5/5.0 이상
  - 예약 프로세스 완료 시간: 평균 3분 이내
- **운영 지표**
  - 웹페이지 초기 로딩: 3초 이내
  - API 응답 시간(P95): 1초 이내
  - 시스템 가용성: 99.5% 이상
- **비즈니스 지표**
  - 월 거래액: 1억원 이상
  - 평균 수수료율: 8~12% 정책 내 유지
  - 정비소 재참여율: 70% 이상

#### 1.3.3 성공 측정 방법
- Google Analytics, CloudWatch, Sentry 기반 실시간 모니터링.
- 주간 운영 리포트 및 월간 KPI 리뷰.
- 분기별 고객/정비소 만족도 조사 및 개선 백로그 반영.

### 1.4 주요 사용자 시나리오

#### 1.4.1 차량 소유자 사용자 여정
**시나리오 1: 예약부터 결제까지**
1. 위치/서비스 조건으로 정비소 검색
2. 정비소 상세 정보(리뷰, 가격 범위, 운영시간) 확인
3. 정비 항목 선택 및 예상 견적 확인
4. 예약 가능 시간 선택 후 차량 정보 입력
5. 예약금 결제 및 예약 확정
6. 정비 완료 후 잔금 결제 및 영수증 수령

**시나리오 2: 진행상황 확인 및 추가 승인**
1. 예약 내역에서 진행 상태 확인
2. 단계별 사진/코멘트 확인
3. 추가 정비 필요 알림 수신
4. 추가 견적 승인 또는 거절
5. 최종 완료 확인 및 차량 인도

#### 1.4.2 정비소 사용자 여정
**시나리오: 예약 접수부터 정산까지**
1. 예약 대시보드에서 신규 예약 확인
2. 입고 접수 후 상태 업데이트 시작
3. 단계별 진행상황과 증빙 사진 등록
4. 추가 작업 발견 시 추가 견적 발송
5. 고객 승인 후 작업 진행
6. 작업 완료 처리 및 결제 완료 확인
7. 매출/정산 내역 조회

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 사용자/인증 시스템
- **회원가입/로그인**
  - 이메일 기반 회원가입 및 로그인 지원
  - JWT 기반 인증 및 세션 관리
  - 소셜 로그인(구글/네이버/카카오) 지원
- **프로필/권한 관리**
  - 사용자 프로필 수정/조회
  - 역할 기반 권한(RBAC) 적용
  - 정비소 직원 계정 등록 및 권한 부여(P1)

### 2.2 차량/정비소 정보 시스템
- **차량 관리**
  - 차량번호, 제조사, 모델, 연식 등 기본 정보 관리
  - 주행거리/특이사항 등 부가 정보 저장
  - 차량별 정비 이력 자동 연결 및 조회(P1)
- **정비소 정보 관리**
  - 정비소 기본 정보, 운영시간, 제공 서비스 등록
  - 서비스별 가격 정책 관리
  - 리뷰/평점/거리 기반 노출 정보 제공

### 2.3 예약/견적 시스템
- **정비소 검색 및 예약 생성**
  - 위치 기반 반경 검색(기본 30km)
  - 필터(서비스, 평점, 가격대) 및 정렬 지원
  - 실시간 예약 가능 슬롯 조회
- **예약 운영**
  - 예약 생성/수정/취소
  - 예약 충돌 방지 로직
  - 예약 리마인더 및 대기 예약(P1)
- **견적 기능**
  - 사전 예상 견적 제공
  - 추가 정비 발견 시 추가 견적 생성
  - 고객의 승인/거절 및 부분 승인 처리(P1)

### 2.4 정비 진행 추적 시스템
- **진행 단계 관리**
  - 접수 -> 진단 -> 작업 -> 검수 -> 완료 단계 관리
  - 단계별 상태 변경 및 예상 완료 시간 업데이트
  - 단계별 코멘트/사진 첨부
- **고객 커뮤니케이션**
  - 진행 단계 변경 시 자동 알림
  - 추가 정비 필요 시 즉시 승인 요청
  - 완료 알림 및 인도 안내

### 2.5 결제/정산 시스템
- **결제 수단**
  - 신용/체크카드, 계좌이체, 휴대폰 결제
  - 카카오페이/네이버페이/토스페이 지원
- **결제 시점**
  - 예약 시 예약금(10~30%) 결제
  - 작업 완료 후 잔금 결제
  - 필요 시 분할 결제 옵션(P1)
- **정산/문서**
  - 영수증/세금 관련 문서 자동 발송
  - 결제 내역 조회 및 환불 처리
  - 정비소 정산 대시보드 제공(P1)

### 2.6 알림/리뷰/운영자 시스템
- **알림 시스템**
  - SMS(기본), 이메일, 웹 푸시(PWA) 지원
  - 예약/진행/완료/추가견적 이벤트 트리거 발송
  - 정기 점검 리마인더 발송(P1)
- **리뷰/평점**
  - 정비 완료 후 1~5점 리뷰 작성
  - 리뷰 신고/관리 및 정비소 답변 기능(P1)
- **관리자 기능**
  - 운영 현황 대시보드
  - 사용자/정비소/분쟁/정산 관리(P1)

### 2.7 기능별 개발 우선순위

#### P0 (필수 - MVP 기능)
- 회원가입/로그인(이메일/소셜)
- 차량 기본 정보 등록
- 정비소 검색/상세 조회
- 예약 생성/조회/수정/취소
- 사전 견적 및 예약금 결제
- 예약 상태 조회
- 이벤트 기반 SMS/이메일 알림

#### P1 (중요 - 초기 출시 후 3개월 내)
- 실시간 진행상황 추적(사진/코멘트 포함)
- 추가 견적 승인 프로세스
- 리뷰/평점 시스템
- 정비 이력 관리
- 관리자 대시보드 및 정산 기능
- 분할 결제/환불 고도화

#### P2 (선택 - 6개월 이후 점진적 추가)
- AI 기반 정비 추천
- 고급 분석/리포팅
- IoT 연동 기반 상태 수집
- 블록체인 기반 정비 이력 신뢰 강화

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능 요구사항
- 웹페이지 초기 로딩: 3초 이내
- API 응답시간(P95): 1초 이내
- DB 쿼리 응답시간(P95): 500ms 이내
- 동시접속자: 최소 100명 지원
- 예약 동시 처리: 10건/초 지원

### 3.2 보안 요구사항
- HTTPS(TLS 1.2+) 통신 필수
- 개인정보 AES-256 암호화 저장
- 비밀번호 bcrypt(salt rounds 12)
- 결제 정보 토큰화 저장
- RBAC 및 Rate Limiting(사용자당 분당 100 요청)
- OWASP Top 10 대응 보안 점검

### 3.3 사용성 요구사항
- 모바일 우선 반응형 설계(320px~1920px)
- 주요 예약 작업 3단계 이내 완료
- WCAG 2.1 AA 수준 접근성 준수
- Chrome/Safari/Edge/Firefox 최신 안정 버전 지원

### 3.4 호환성 및 확장성 요구사항
- PWA 지원(Android/iOS 브라우저 환경)
- 마이크로서비스 전환 가능한 모듈형 설계
- 수평 확장, 로드밸런싱, Stateless 서비스 구조
- Read Replica/캐시 기반 트래픽 확장 지원

### 3.5 데이터 관리 요구사항
- DB 일일 자동 백업 + 월별 아카이브
- RTO 4시간, RPO 1시간
- 개인정보 탈퇴 즉시 삭제 정책
- 거래 데이터 7년 보관, 로그 1년 보관

### 6.1 회원/인증 기능
**승인 조건:**
- [ ] 이메일 회원가입/로그인이 정상 동작한다.
- [ ] 소셜 로그인(구글/네이버/카카오) 3종이 정상 동작한다.
- [ ] JWT 토큰 만료/재인증 플로우가 정상 동작한다.
- [ ] 역할별 접근 권한이 분리되어 동작한다.
- [ ] 비밀번호 및 개인정보 저장 정책이 보안 요구사항을 만족한다.

### 6.2 예약/검색 시스템
**승인 조건:**
- [ ] 위치/필터 기반 정비소 검색 결과가 정확히 표시된다.
- [ ] 예약 가능 시간 슬롯이 실시간으로 노출된다.
- [ ] 동일 슬롯 중복 예약이 차단된다.
- [ ] 예약 생성/변경/취소가 정책에 따라 처리된다.
- [ ] 예약 상태 조회 화면에서 최신 상태를 확인할 수 있다.

### 6.3 견적/결제 시스템
**승인 조건:**
- [ ] 사전 견적 금액과 항목이 예약 전에 확인 가능하다.
- [ ] 예약금 결제가 정상 승인/실패 처리된다.
- [ ] 잔금 결제 및 영수증 발급이 정상 처리된다.
- [ ] 간편결제 3종(카카오/네이버/토스) 중 최소 1종 이상이 MVP에서 동작한다.
- [ ] 취소/환불 시 결제 상태와 금액이 일관되게 반영된다.

### 6.4 진행상황/알림 시스템
**승인 조건:**
- [ ] 단계별 진행상황 업데이트가 고객 화면에 반영된다.
- [ ] 단계 변경 이벤트 시 SMS 또는 이메일 알림이 발송된다.
- [ ] 추가 정비 요청 시 고객 승인/거절 결과가 즉시 반영된다.
- [ ] 알림 발송 실패 시 재시도 정책이 동작한다.
- [ ] 완료 알림과 차량 인도 안내가 정상 발송된다.

### 6.5 운영/관리 기능
**승인 조건:**
- [ ] 운영 대시보드에서 핵심 지표(MAU, 예약, 거래액)를 조회할 수 있다.
- [ ] 사용자/정비소 계정 상태를 관리할 수 있다.
- [ ] 분쟁/신고 처리 이력이 기록되고 조회된다.
- [ ] 정산 내역 조회 및 기간별 집계가 가능하다.
- [ ] 감사 로그가 추적 가능하게 저장된다.

## 7. 테스트 전략

### 7.1 테스트 유형별 전략
**단위 테스트**
- 핵심 비즈니스 로직 단위 테스트 작성
- 서비스 레이어 기준 80% 이상 커버리지 목표

**통합 테스트**
- DB 연동, 외부 API(결제/지도/알림) 연동 테스트
- 예약 생성->진행->결제 완료 E2E 시나리오 검증

**성능 테스트**
- 동시 사용자 100명 부하 테스트
- API 응답 시간 SLA(1초) 검증

**보안 테스트**
- OWASP Top 10 취약점 점검
- SQL Injection, XSS, 인증/인가 우회 테스트
